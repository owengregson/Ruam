# JS AST Builder — Runtime Template System Redesign

## Problem

The VM runtime is generated by 11 TypeScript files totaling 2,344 lines. Roughly 1,200 of those lines are JavaScript source code embedded inside template literal strings. This causes three problems:

1. **Authoring** — No syntax highlighting, autocomplete, or type checking on the generated JS. Typos become runtime failures.
2. **Post-processing** — Five regex-based passes (inline stack ops, obfuscate locals, filter unused handlers, inject decoys, remap case labels) depend on exact formatting and fail silently when it changes.
3. **Structure** — `interpreter.ts` is a 1,147-line megafile mixing code generation, post-processing, and all 316 opcode handlers in one function.

## Solution

Replace string-based code generation with a purpose-built JavaScript AST builder. Templates become TypeScript functions that construct AST nodes. Post-processing becomes tree transformations. The emitter serializes the tree to minified JS in one pass.

## AST Node System

~30 node types covering the JS constructs the runtime templates use. Factory functions have short names since they're called thousands of times.

### Node Types

| Category | Nodes |
|----------|-------|
| Declarations | `VarDecl`, `ConstDecl`, `FnDecl`, `ParamList` |
| Statements | `Block`, `If`, `While`, `For`, `ForIn`, `Switch`, `Case`, `Break`, `Continue`, `Return`, `Throw`, `TryCatch` |
| Expressions | `Id`, `Literal`, `BinOp`, `UnaryOp`, `Assign`, `Call`, `MemberExpr`, `Index`, `Ternary`, `Array`, `Object`, `FnExpr`, `ArrowFn`, `New`, `Typeof`, `Sequence` |
| Special | `Raw` — escape hatch for injecting raw JS strings verbatim |

File: `src/codegen/nodes.ts` (~80 lines)

## Emitter

Single recursive function: `emit(node): string`. Produces minified JS output.

- Minified by default. No pretty-printing option. The AST is the readable representation.
- Automatic semicolons after statements.
- Precedence-aware parenthesization — only adds parens when operator precedence requires them.
- `Raw` nodes emit verbatim.

File: `src/codegen/emit.ts` (~200 lines)

## Tree Transformations

The 5 current regex-based post-processing passes become structural tree operations:

| Current (regex) | New (tree operation) |
|-----------------|---------------------|
| `inlineStackOps()` — paren-counting string replacement | Visitor: pattern-match `Call(id(W), [expr])` → `Assign(Index(S, PreInc(P)), expr)`. Remove W/X/Y function declarations. |
| `obfuscateLocals()` — regex `\bvar\s+([a-zA-Z]...)` | Visitor: collect VarDecl names 3+ chars not in KEEP set, generate short replacements via LCG, rename all matching Id references. |
| `filterUnusedOpcodeHandlers()` — complex case-block regex | Filter: `cases.filter(c => usedOpcodes.has(c.label))` |
| `injectDecoyHandlers()` — insert before `default:break;` | Push: append decoy `CaseClause` nodes to the cases array. |
| Case label remapping — regex `\bcase (\d+):` | Map: `caseClause(shuffleMap[op], handler(ctx))` during construction. |

File: `src/codegen/transforms.ts` (~250 lines)

## Opcode Handler Registry

Each of the ~316 opcode handlers becomes a standalone function returning `JsNode[]`. Handlers are grouped by category into separate files.

### HandlerCtx

```typescript
interface HandlerCtx {
  S: string;    // stack array
  P: string;    // stack pointer
  IP: string;   // instruction pointer
  C: string;    // constants array
  O: string;    // operand
  SC: string;   // scope
  R: string;    // registers
  EX: string;   // exception handler stack
  PE: string;   // pending exception
  HPE: string;  // has pending exception
  CT: string;   // completion type
  CV: string;   // completion value
  // ... ~15 fields total, derived from RuntimeNames
}
```

### Interpreter Builder

```typescript
function buildInterpreterSwitch(ctx: HandlerCtx, usedOpcodes?: Set<number>): JsNode {
  const cases = [];
  for (const [op, handler] of registry) {
    if (usedOpcodes && !usedOpcodes.has(op)) continue;
    cases.push(caseClause(shuffleMap[op], handler(ctx)));
  }
  return switchStmt(id(ctx.PH), cases);
}
```

Dynamic opcode filtering is a `continue`. Decoy injection is pushing extra nodes. Case label remapping is `shuffleMap[op]` at construction time.

File: `src/codegen/builders/interpreter.ts` (~100 lines, assembles HandlerCtx + switch + wrapping function declaration)

## Template Builders

The remaining templates (loader, runners, debug protection, etc.) become builder functions returning `JsNode[]`:

```
src/codegen/builders/
  loader.ts, runners.ts, globals.ts, deserializer.ts,
  debug-logging.ts, debug-protection.ts, rolling-cipher.ts,
  decoder.ts, fingerprint.ts, stack-encoding.ts
```

The orchestrator (`vm.ts`) assembles AST nodes from all builders and calls `emit()` once:

```typescript
const nodes: JsNode[] = [];
if (encrypt) nodes.push(...generateFingerprint(names), ...generateDecoder(names));
if (dbgProt) nodes.push(...generateDebugProtection(names));
// ... all builders ...
return emit(iife(block(...nodes)));
```

## Deleted After Migration

- `src/runtime/templates/` — entire directory (7 files)
- `generate*Source()` functions from `src/runtime/rolling-cipher.ts`, `decoder.ts`, `fingerprint.ts`
- `obfuscateLocals()`, `inlineStackOps()`, `filterUnusedOpcodeHandlers()`, `injectDecoyHandlers()` from old `interpreter.ts`

## Unchanged

- `src/runtime/names.ts` — RuntimeNames interface and generators
- `src/runtime/vm.ts` — orchestrator logic (switches from string concat to AST assembly + `emit()`)
- `src/compiler/` — entirely untouched
- `src/transform.ts` — untouched
- All existing tests — output is functionally identical

## Migration Order

1. **Core infrastructure** — `nodes.ts`, `emit.ts`, `transforms.ts` + emitter unit tests
2. **Opcode handlers** — extract ~316 handlers into `handlers/` files, build registry, write `interpreter.ts` builder. Use `raw()` for non-interpreter templates.
3. **Small builders** — `globals.ts`, `fingerprint.ts`, `rolling-cipher.ts`, `decoder.ts`
4. **Medium builders** — `loader.ts`, `runners.ts`, `deserializer.ts`, `stack-encoding.ts`
5. **Complex builders** — `debug-protection.ts`, `debug-logging.ts`
6. **Orchestrator** — update `vm.ts` to assemble AST nodes and call `emit()` once
7. **Delete old templates** — remove `src/runtime/templates/` and migrated functions

Each step is independently shippable. Tests pass after every step. The `raw()` escape hatch allows unconverted templates to coexist with converted ones.
